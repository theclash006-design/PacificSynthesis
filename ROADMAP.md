# Pacific Synthesis — 配布までのロードマップ

> 最終目標: 商業配布で収益化
> 戦略: A → B → C → D の順に踏む。いきなり D は実績ゼロで失敗する

---

## 公開レベルの定義

| 段階 | 必要なもの | 想定収益 |
|---|---|---|
| **A. GitHub 無料公開** | コード + README | 0円 (実績作り) |
| **B. 自サイトで無料配布** | A + Apple Developer ($99/年) + 署名 + Notarize | 0円〜寄付 |
| **C. 個人サイトで有料販売** | B + 決済 (Gumroad/Stripe) + デモ音源 + 体験版 | 月数万〜数十万 |
| **D. Plugin Boutique 等で販売** | C + 審査通過 + マーケティング | 月数十万〜 |

## 実行プラン

### Phase 1 (1-2 週間) — A 段階の準備
1. **コードを GitHub 公開できる状態に整える**
   - README (ビルド手順、使い方、スクリーンショット)
   - LICENSE (推奨: MIT or GPLv3 — どちらにするか後で議論)
   - .gitignore 整備
2. **音の客観評価を取る**
   - 音楽好きな友達 2-3 人に「これで1曲作ってみて」依頼
   - フィードバック収集
3. **クラッシュテスト**
   - 何時間も連続再生
   - 複数 DAW (Logic / Ableton / Cubase / Reaper) でロード
   - 極端なパラメータで壊れないか

### Phase 2 (1-3 ヶ月) — B 段階
4. RT 安全性監査と DSP 改良
5. Apple Developer 登録、署名・公証スクリプト作成
6. 署名付き .pkg で配布開始
7. Gumroad で無料公開してメールリスト集める
8. デモトラックを SoundCloud / YouTube に公開

### Phase 3 (その先) — C 段階
9. ファンが付いたら有料版 (v2.0?) を企画
10. 体験版 + 製品版の2本立て
11. 価格は最初 $19-29 が現実的
12. アップデートで継続的に改善
