# Pacific Synthesis VST3 ビルド手順 (macOS)

このプロジェクトは **JUCE 8 + CMake** で組まれた VST3 / AU プラグインのスタータープロジェクトです。

---

## 必要環境

- macOS 10.15 以降
- **Xcode 14+** (App Store からインストール、初回起動でコマンドラインツールも入る)
- **CMake 3.22+** (`brew install cmake`)
- **Git** (Xcode付属のもので OK)

---

## 1. JUCE を取得

このプロジェクトの **親ディレクトリ** に JUCE をクローンします。

```bash
cd ~/Downloads
git clone --depth 1 https://github.com/juce-framework/JUCE
```

結果として以下の配置になっていれば OK：

```
~/Downloads/
  ├── JUCE/                       ← 今クローンしたもの
  └── PacificSynthesisVST3/
      ├── CMakeLists.txt
      ├── Source/
      ├── 01_移植プラン.md
      └── 02_ビルド手順.md
```

別の場所に置きたい場合は、後述の CMake コマンドに `-DJUCE_PATH=/path/to/JUCE` を追加してください。

---

## 2. Xcode プロジェクトを生成

```bash
cd ~/Downloads/PacificSynthesisVST3
cmake -B build -G Xcode
```

`build/PacificSynthesis.xcodeproj` が生成されます。

---

## 3. ビルド

### 方法A: Xcode GUI で開く

```bash
open build/PacificSynthesis.xcodeproj
```

- スキーム を **PacificSynthesis_VST3** （または `_AU`, `_Standalone`）に切り替え
- Cmd+B でビルド

### 方法B: コマンドラインだけで完結

```bash
cmake --build build --config Release
```

ビルド成功時、以下に成果物が出ます（JUCE の `COPY_PLUGIN_AFTER_BUILD` で自動コピーされます）：

| 形式 | コピー先 |
|------|---------|
| VST3 | `~/Library/Audio/Plug-Ins/VST3/Pacific Synthesis.vst3` |
| AU   | `~/Library/Audio/Plug-Ins/Components/Pacific Synthesis.component` |
| Standalone | `build/PacificSynthesis_artefacts/Release/Standalone/Pacific Synthesis.app` |

---

## 4. DAW で使う

### Logic Pro / GarageBand
- AU のみ。初回起動時に自動スキャンされる
- Instrument タブ → AU Instruments → Coast Machine → Pacific Synthesis

### Ableton Live
- 環境設定 → Plug-ins → Use VST3 Plug-in System Folders を ON
- ブラウザの Plug-Ins から Pacific Synthesis をトラックにドラッグ

### Cubase / Studio One / Bitwig
- VST3 スキャンに任せれば自動で出る

---

## 5. MIDI 入力の使い方

| ノート | 音 |
|--------|-----|
| C1 (note 36) | Kick |
| D1 (note 38) | Snare |
| D#1 (note 39) | Clap |
| F#1 (note 42) | HiHat |
| その他 | Bass |

- **Velocity ≥ 100** で Accent モード（フィルタが大きく開く）
- ベースで **ノートを重ねる** とスライド（前のノートが鳴っている間に新ノート）

DAW のステップシーケンサーや MIDI クリップで C1〜F#1 にドラム、C2以上にベースラインを置いてください。

## 6. トラブルシューティング

### `JUCE not found at ...` エラー
JUCE が想定の場所にありません。コマンドにパスを渡してください：
```bash
cmake -B build -G Xcode -DJUCE_PATH=/your/path/to/JUCE
```

### Logic Pro でプラグインが出ない
- macOS のセキュリティで AU が弾かれている可能性があります
- 一度 `auval -v aumu Pcfs Csmc` を実行してバリデーション結果を確認
- 必要に応じて Logic を再起動

### ビルドが C++17 でこける
- CMake のキャッシュをクリア: `rm -rf build && cmake -B build -G Xcode`

### `LadderFilter` 周りのコンパイルエラー
- JUCE 8 系を使っているか確認 (`cd ~/Downloads/JUCE && git describe --tags`)
- 古い JUCE 6 系だと一部 API が違います

## 7. 次にやれること（拡張アイデア）

1. **内蔵 16ステップシーケンサー** をホストの再生に同期して動かす（元 HTML 完全再現）
2. **Buchla 風 GUI** ：SVG ノブと cream/navy パレットを LookAndFeel で実装
3. **MIDI ファイルエクスポート** 機能
4. **プリセット保存/読み込み** （元 HTML の Pattern Bank 相当）
5. **より忠実な 303 モデリング**：MS-20 風 SVF と入れ替え、または diode ladder
