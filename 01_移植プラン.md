# PACIFIC SYNTHESIS → VST3 (Mac) 移植プラン

> 元ファイル: `pacific-synthesis_3.html`
> ターゲット: macOS / VST3 (AUも同時ビルド可)
> 推奨フレームワーク: **JUCE 8 + CMake**

---

## 1. 元シンセの構造（コードリーディング結果）

`pacific-synthesis_3.html` は React + Tone.js でできた **16ステップのドラム＋303風ベースシーケンサー**です。
オーディオ部分は全部 Tone.js（つまりWeb Audio API）のラッパーで組まれていて、
DSP的にはそこまで複雑ではありません。要素を整理するとこうなります。

### 1.1 ドラム（4トラック）

| トラック | 音源 | パラメータ |
|---------|---------|-----------|
| KICK   | `MembraneSynth` (sine, pitchDecay 0.05, 6oct) | decay 0.4 |
| SNARE  | `NoiseSynth` (white) + 小さい `MembraneSynth` (E2) のレイヤー | decay 0.15 |
| HIHAT  | `NoiseSynth` (white) → `Filter` 8kHz bandpass → ディレイsend | decay 0.06 |
| CLAP   | `NoiseSynth` (pink) → `Filter` 1.2kHz bandpass → ディレイsend | decay 0.12 |

すべて短いAR（attack 0.001s）。Tone.jsの `MembraneSynth` は内部で「サイン波 + ピッチエンベロープ + アンプエンベロープ」なので、自前で簡単に再現可能。

### 1.2 ベース（303風）

```
MonoSynth (saw/square)
  → Filter (-24dB/oct LP, Q可変, freq cutoff)
  → FeedbackDelay (8n.) [send]
  → Reverb (decay 1.5s) [send]
  → Distortion (waveshape) [send]
  → Master Gain
```

特徴的なのが **accent (アクセント)**:
- アクセント時に `cutoff + envMod * accAmt` まで一瞬持ち上げ、`bassDec` 秒かけて元のcutoffに戻る
- これが TB-303 のキモ "filter envelope kick" の挙動

**slide (スライド)**: スライド時はノート長を `16n` → `8n` に伸ばしてレガート風に。
（厳密な303ポルタメントではないが、聴感上スライド気味になる）

### 1.3 シーケンサー

- 16ステップ、`Tone.Sequence` で `16n` グリッド
- BPM (60–200)、Swing (0–1)、各ステップ on/off
- ベース側は **ノート名 + オクターブ + accent + slide**
- LocalStorage に最大8パターン保存
- MIDIファイルエクスポート (PPQ 480) もあり

### 1.4 公開パラメータ一覧

| パラメータ | 範囲 | 単位 | デフォルト | VST3パラメータID候補 |
|-----------|------|------|-----------|---------------------|
| BPM       | 60–200 | bpm | 128 | `bpm` (※ホスト同期可) |
| SWING     | 0–1 | – | 0 | `swing` |
| VOL       | 0–1 | – | 0.7 | `masterGain` |
| WAVE      | saw/sqr | – | saw | `bassWave` |
| CUTOFF    | 100–5000 | Hz | 800 | `cutoff` |
| RESO (Q)  | 0–20 | – | 8 | `reso` |
| ENV MOD   | 0–6000 | Hz | 3000 | `envMod` |
| DECAY     | 0.05–1.5 | s | 0.3 | `bassDec` |
| ACCENT    | 0–1 | – | 0.5 | `accAmt` |
| DELAY     | 0–0.6 | mix | 0.15 | `delMix` |
| REVERB    | 0–0.6 | mix | 0.1 | `revMix` |
| DIST      | 0–1 | – | 0 | `dist` |

## 2. アプローチの比較と推奨

| 手法 | 工数 | 音質再現度 | 安定性 | DAW互換 | 評価 |
|------|------|-----------|--------|---------|------|
| **JUCE (C++) で書き直し** | 中〜大 | ◎ | ◎ | ◎ VST3/AU/AAX全部 | **★ 推奨** |
| Camomile (Pd → VST3)     | 中    | ○ | ○ | ○ VST3のみ | 簡単だがGUI制約 |
| WAM (Web Audio Modules)  | 小    | ◎ (完全同一) | △ | △ Bitwig/一部のみ | DAW互換が弱い |
| iPlug2                   | 中    | ◎ | ○ | ◎ | OK だがMac特化ならJUCE優位 |

**今回の推奨は JUCE 8 + CMake** です。理由：

1. Tone.js のオブジェクトはほぼ全部 JUCE DSPモジュール (`dsp::IIR::Filter`, `dsp::Oscillator`, `Reverb`, `WaveShaper`) で 1:1 対応できる
2. macOSで VST3 と AU を同じソースから同時ビルドできる
3. Logic / Cubase / Ableton すべてで問題なく動く
4. Xcodeデバッグも快適

## 3. 移植方針 — 「何を残し、何を捨てるか」

Web版の機能をすべてVST3に持ち込むと工数が爆発するので、優先度を分けます。

### フェーズ1（このプロジェクトで提供する初期版）

- ✅ 4ドラム＋ベースの **音源部**
- ✅ MIDI入力対応（ホストのシーケンサーで鳴らす）
  - C1=Kick, D1=Snare, F#1=HiHat, D#1=Clap （GM準拠）
  - C2-C5 がベース
- ✅ 全パラメータをVST3パラメータとして公開
- ✅ ベースのアクセント (Velocity ≥ 100 → accent), スライド (overlapping notes → slide)
- ✅ ディレイ/リバーブ/ディストーション
- ✅ シンプルなGUI（パラメータをノブで配置）

### フェーズ2（後で足す）

- ⏳ 内蔵16ステップシーケンサー（ホストの再生に同期）
- ⏳ パターンのプリセット管理（VSTプリセット）
- ⏳ Buchla風GUIの完全再現（Cream/Navyカラーパレット、SVGノブ）
- ⏳ MIDIエクスポート

このプランの理由は「**まず音を出せる状態にする**」が最優先で、GUIや内蔵シーケンサーは後からインクリメンタルに足せるからです。

## 5. 次のステップ

1. JUCEをローカルに取得（`git clone https://github.com/juce-framework/JUCE`）
2. このフォルダ内で `cmake -B build -G Xcode` 実行
3. Xcodeで開いてビルド → `~/Library/Audio/Plug-Ins/VST3/PacificSynthesis.vst3` が生成
4. Logic / Ableton / Cubase で読み込み

詳細は `02_ビルド手順.md` を参照してください。
